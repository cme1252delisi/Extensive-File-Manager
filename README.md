# 🚀 Extensive File Manager (EFM)
> A high-performance, CLI-driven file management ecosystem built with Java.

**Extensive File Manager (EFM)** is designed for power users who demand absolute, low-level control over their file system directly from the terminal. By completely stripping away GUI overhead, EFM delivers native-speed operations, seamless cryptography, and advanced archiving—all under a single, modular Command Line Interface.

## ✨ Key Features

* **⚡ Blazing Fast Operations:** Zero visual rendering. Zero mouse interactions. Pure keyboard-driven speed for high-complexity file tasks.
* **🛠️ Unified Workspace:** Forget switching between disparate tools. Edit contents, encrypt with AES, compress to ZIP/TAR, and convert formats without ever leaving your terminal session.
* **🌍 Write Once, Run Anywhere:** Built on Java's robust NIO.2 libraries, ensuring identical command syntax and perfect compatibility across Linux, macOS, and Windows file systems (ext4, NTFS, FAT32).
* **🧩 Layered & Extensible Architecture:** Core file system operations are strictly isolated from high-level user commands, making it incredibly easy to integrate new file systems or encryption algorithms.

## 🎯 Why EFM?

We built EFM to solve the modern developer's context-switching fatigue. When you are deep in a terminal environment, forcing yourself to open a heavy GUI manager to handle archives or encrypt files breaks your workflow. EFM operates with a minimal CPU/RAM footprint, making it perfect for headless server environments, legacy hardware, or developers who just want to stay in the zone.
